# ARX Sheet Step 1 — Base Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Roll20 legacy character sheet base page (level, 4 attributes, 9 skills, health/mana, damages, 3 resistances) rendered over the ARX-BASE.png grimoire image, produced by a Python build pipeline.

**Architecture:** Jinja2 templates + plain CSS sources in `src/`, assembled by `build.py` into the two Roll20 deliverables `build/sheet.html` (fragment) and `build/sheet.css`, plus a local `build/preview.html` for browser iteration. The composed image is the background of a fixed-size container; every field is an absolutely positioned transparent `<input type="number">` placed below its engraved icon.

**Tech Stack:** Python 3.12, Jinja2, pytest. No Node, no Sass, no sheet workers (step 1 has no calculations).

**Spec:** `docs/superpowers/specs/2026-07-20-base-page-design.md` (approved).

## Global Constraints

- Roll20 legacy sheet: `sheet.html` is an HTML **fragment** — no `<html>`, `<head>`, `<body>`, `<script src>`, `<iframe>`, no inline `style=` attributes (Roll20 strips them).
- All data fields are inputs named `attr_<name>`, lowercase snake_case, English names.
- All CSS classes are written **already prefixed** with `sheet-` (e.g. `class="sheet-field"`), so selectors work identically in Roll20's legacy and CSE editors.
- Images only via HTTPS: `https://raw.githubusercontent.com/C-Chafik/arx-jdr-sheet/main/assets` (repo remote already exists: `git@github.com:C-Chafik/arx-jdr-sheet.git`).
- Reference image: `assets/ARX-BASE.png`, **1575 × 999 px** (final re-export, "Xp" removed). Container is 1100 px wide → height 698 px (same ratio).
- No dice rolls, no auto-calculations, no identity fields, no ring slots (all deferred).
- Minimal code: no speculative options, no config files beyond `requirements.txt`.

- [ ] **Step 6: Write `build.py`**

```python
#!/usr/bin/env python3
"""Assemble the Roll20 sheet: render Jinja2 templates, concatenate CSS."""
import argparse
import time
from pathlib import Path

from jinja2 import Environment, FileSystemLoader

ROOT = Path(__file__).parent
SRC = ROOT / "src"
BUILD = ROOT / "build"

# Concatenation order matters: later files override earlier ones.
CSS_FILES = ["base.css", "base-page.css"]

# Roll20 only loads HTTPS assets; preview uses the local files instead.
ASSET_BASE = "https://raw.githubusercontent.com/C-Chafik/arx-jdr-sheet/main/assets"
PREVIEW_ASSET_BASE = "../assets"

PREVIEW_WRAPPER = """<!doctype html>
<html>
<head>
<meta charset="utf-8">
<title>ARX sheet preview</title>
<link rel="stylesheet" href="preview.css">
</head>
<body>
{content}
</body>
</html>
"""


def render_html() -> str:
    env = Environment(loader=FileSystemLoader(SRC / "templates"), keep_trailing_newline=True)
    return env.get_template("sheet.html.j2").render()


def build_css(asset_base: str) -> str:
    parts = [(SRC / "css" / name).read_text() for name in CSS_FILES]
    return "\n".join(parts).replace("{{ASSET_BASE}}", asset_base)


def build() -> None:
    BUILD.mkdir(exist_ok=True)
    html = render_html()
    (BUILD / "sheet.html").write_text(html)
    (BUILD / "sheet.css").write_text(build_css(ASSET_BASE))
    (BUILD / "preview.css").write_text(build_css(PREVIEW_ASSET_BASE))
    (BUILD / "preview.html").write_text(PREVIEW_WRAPPER.format(content=html))
    print("built: build/sheet.html sheet.css preview.html preview.css")


def watch() -> None:
    last = None
    while True:
        current = {p: p.stat().st_mtime for p in SRC.rglob("*") if p.is_file()}
        if current != last:
            build()
            last = current
        time.sleep(0.5)


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--watch", action="store_true")
    args = parser.parse_args()
    (watch if args.watch else build)()
```

## Self-Review Notes

- **Spec coverage:** build pipeline (Task 1), 22 fields incl. `_max` pairs (Task 2), visual layer + positioning + font (Task 3), HTTPS hosting + sandbox verification incl. token-bar check (Task 4). Ring slots, rolls, workers: explicitly out of scope per spec.
- **Deviation from spec, deliberate:** positions live in handwritten CSS (`base-page.css`) rather than a Jinja `(attr_name, position)` list — Roll20 strips inline `style=` attributes, so positions must be CSS classes anyway; one source of truth.
- **Type consistency:** class names derive from the same Jinja lists that name the attributes; the tests cross-check HTML attrs (Task 2) and CSS classes (Task 3) against the same `ATTRS` list.
